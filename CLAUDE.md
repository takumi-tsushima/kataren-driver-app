# プロジェクト概要
カタレン向け ドライバー管理用WEBアプリ

---

# 目的
回送ドライバーの管理業務を一元化するためのアプリ。

現在は以下が分散している：
- シフト管理：スプレッドシート
- 案件募集：Slack
- 請求管理：Invoy

これらを統合し、ドライバー用マイページ＋管理画面で完結させる。

---

# 技術スタック
- フロントエンド：React + TypeScript + Vite
- UI：モバイルファースト（SP優先）
- バックエンド：Supabase（Auth / PostgreSQL）
- デプロイ：Vercel

# シフト仕様（重要）
- stateは Map<string, Shift> で管理
- Shift型：
  {
    date: string
    status: 'ok' | 'ng' | 'none'
    timeSlot?: string
    maxJobs?: number
    note?: string
  }

- status定義：
  - ok：稼働可能
  - ng：稼働不可
  - none：未入力（保存しない / DB削除対象）

- 今日から7日以内は編集不可

- 時間は startTime ではなく timeSlot（例："9:00-12:00"）で管理

# Supabase設計方針
- RLS前提
- 最小権限
- driverは自分のデータのみアクセス可能
- adminは全データ操作可能
